# 22_TrailBlazers
---
**Problem statement:**  
**Event Execution and Post - Event Analytics Platform**  
---

## Overview :
In real-world events, clients depend on event managers and on-site coordinators to execute events smoothly, but visibility into attendee participation, session engagement, and operational issues is often fragmented or informal.

Engagement data such as RSVPs, actual attendance, session participation, and time spent at the event is usually tracked manually or not consolidated in one place.

On-site activities like check-ins, issue handling, and attendee movement are managed separately, making it difficult to relate execution quality to event outcomes.

After the event, event managers must manually interpret participation trends and feedback to report event success to clients.

There is a need for a structured way to support event execution and post-event evaluation based on how events are actually managed in practice.

---

## Identified actors :

| Actor / Role   | Description                                                                  |
|---------------|------------------------------------------------------------------------------|
| **Client / Organization**         | The party that requests and funds an event; defines high-level objectives and reviews post-event outcomes|
| **Event Manager**         | Responsible for planning, coordinating, and delivering the event end-to-end, and preparing post-event reports      |
| **On - Site Event Co - Ordinator** | Handles on-ground execution, check-ins, crowd flow, and immediate operational issues during the event        |
| **Administrator**     | Platform/system actor who manages requests, assignments, access control, and overall platform governance         |
| **Attendee**    | Participant of the event who RSVPs to an event, attends sessions, engages with content, and provides feedback     |

- Register on the platform
- Request an event
- Provide event requirements to the assigned event manager
- Review post-event analytics
- View consolidated attendee feedback

## Event Manager
- Structure attendee engagement strategies (polls, Q&A, networking)
- Monitor live event execution
- Track RSVP versus actual attendance
- Analyze session-wise attendee engagement
- Appraising event success metrics
- Review on-site issues escalated during the event
- Prepare post-event performance report
- Submit event report to the client
 
---
 
## Attendee
- RSVP to an event
- Check in at the event venue
- Engage with like minded individuals
- Provide feedback during or after the event
 
---
 
## On-site Event Coordinator / Supervisor
- Verify attendee check-in
- Manage entry, exit, and crowd flow
- Coordinate session transitions
- Work with the speakers closely
- Identify and report operational issues
- Escalate issues to the event manager
